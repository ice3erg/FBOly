# Инструкция пользователя

## 1. Запуск без Docker

Основной способ запуска:

```text
start-portable.bat
```

Откроется окно сервера. Не закрывайте его, пока работаете с сервисом.

Затем откройте:

```text
http://localhost:3000
```

Если скрипт пишет `Node.js not found`, установите Node.js LTS:

```text
https://nodejs.org/
```

После установки снова запустите `start-portable.bat`.

Чтобы остановить portable-режим, закройте окно сервера или запустите:

```text
stop-portable.bat
```

## 2. Запуск через Docker

```text
start-service.bat
```

Docker-режим нужен только если Docker Desktop нормально запускается. Если Docker не грузится, используйте `start-portable.bat`.

Если запускаете Docker-режим из PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -File .\start-service.ps1
```

Когда сервис запустится, откройте в браузере:

```text
http://localhost:3000
```

## 3. Работа в сервисе

1. Введите `Client-Id` и `Api-Key` из Ozon Seller.
2. Поставьте галочку `Сохранить API в этом браузере`, если хотите не вводить ключи каждый раз.
3. Нажмите `Проверить подключение`.
4. Нажмите `Загрузить склады Ozon`, чтобы сервис увидел доступные FBO-склады и кластеры Ozon.
5. Загрузите Excel-файл с товарами.
6. Нажмите `Создать файлы`.
7. Скачайте общий ZIP или отдельные XLSX-файлы по складам.

Если введены Ozon API ключи, сервис распределяет товары автоматически:

- смотрит SKU/артикулы из загруженного Excel;
- получает из Ozon список доступных FBO-складов/кластеров через `/v1/cluster/list`;
- получает по товарам данные Ozon;
- учитывает продажи, оборачиваемость и текущие остатки;
- больше отправляет туда, где спрос выше и остатка меньше;
- меньше отправляет туда, где спрос ниже или товар уже есть на остатках.
- если по складу нет потребности, этот склад пропускается и файл по нему не создаётся.
- технические склады и хабы Ozon объединяются в городские кластеры: например все `КАЗАНЬ_*` попадают в `Казань`, все `СПБ_*` в `Санкт-Петербург`.

Если в вашем файле Ozon SKU находится в колонке `артикул`, сервис тоже попробует найти товар по этому значению как по SKU Ozon.

Для новых товаров без истории продаж сервис распределяет поставку только по крупным складам:

```text
Москва -> Санкт-Петербург -> Казань -> Ростов -> Екатеринбург -> Воронеж
```

Когда по товару появятся продажи, сервис начнёт распределять его по всем складам/регионам, где Ozon показывает спрос.

Тестовый файл для проверки:

```text
examples/test_products.xlsx
```

Если Ozon не отдаёт аналитику или временно ограничивает запросы, сервис использует проценты складов как запасной вариант.

## Роли Ozon API

Для этого сервиса лучше выбрать роль:

```text
Admin read only
```

Она даёт доступ ко всем информационным методам Seller API без права что-либо менять. Это безопаснее, чем `Admin`, потому что сервис только читает данные Ozon и создаёт Excel-файлы локально.

Если хотите собрать ключ из отдельных ролей, выберите минимум:

- `Product read-only` — получить информацию о товарах, SKU и артикулах;
- `Warehouse` — получить данные по складам и остаткам;
- `Report` — получить аналитику/отчёты;
- `Supply order ReadOnly` — читать данные по FBO-поставкам.

Для будущего автоматического создания поставок через API понадобится уже `Supply order`, но для текущего MVP это не нужно.

## 4. Входной Excel

В portable-режиме нужен файл `.xlsx`.

В файле могут быть такие колонки:

- `SKU Ozon`
- `артикул`
- `название товара`
- `количество`

Если есть `артикул`, сервис сразу использует его для выходного файла.

Если есть только `SKU Ozon` или `название товара`, сервис ищет артикул через Ozon Seller API.

## 5. Выходные файлы

Сервис создаёт:

- `Ozon_FBO_Москва_Хоругвино.xlsx`
- `Ozon_FBO_Санкт-Петербург.xlsx`
- `Ozon_FBO_Казань.xlsx`

В каждом файле строго 3 колонки:

```text
артикул
имя (необязательно)
количество
```

## 6. Если товар не найден

Такая строка появится в блоке `Ошибки поиска`.

Проверьте:

- правильность SKU;
- название товара;
- доступность товара в кабинете продавца Ozon;
- корректность `Client-Id` и `Api-Key`.

## 7. Остановка сервиса

Запустите:

```text
stop-service.bat
```

Или в PowerShell:

```powershell
.\stop-service.ps1
```

## 8. Если сервис не запускается

### PowerShell пишет, что `start-service.ps1` не найден

Вы находитесь не в папке проекта. Сначала выполните:

```powershell
cd "C:\Users\princ\Documents\Codex\2026-05-12\mvp-ozon-fbo-excel-sku-ozon"
```

Потом:

```powershell
.\start-service.bat
```

### PowerShell пишет, что выполнение сценариев отключено

Запустите `.bat`:

```powershell
.\start-service.bat
```

Или так:

```powershell
powershell -ExecutionPolicy Bypass -File .\start-service.ps1
```

### Скрипт пишет `Docker Desktop Service is not running`

Сделайте один раз:

1. Закройте Docker Desktop через значок в трее.
2. Откройте Docker Desktop от имени администратора.
3. Дождитесь статуса `Engine running`.
4. Снова запустите `start-service.bat`.

Альтернатива: откройте PowerShell от имени администратора и выполните:

```powershell
Start-Service com.docker.service
```

Затем вернитесь в папку проекта и запустите:

```powershell
.\start-service.bat
```
